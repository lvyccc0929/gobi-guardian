# 戈壁守望者 · GUARDIAN OF THE GOBI## 项目概况基于 2025 年真实事件改编的 H5 互动新闻叙事实验。事件背景：新疆哈密戈壁深处，1967 年空军学员用石块铺设的巨型大地标语「为人民服务」遭越野车碾压破坏。一位视频博主通过卫星地图发现异常后，跨越 2000 公里奔赴现场，寻找匹配的石料，自发完成修复。本作品将这一事件转化为 8 幕移动端互动体验。全部代码由 AI 编程助手（Codex CLI）生成，我负责选题策划、事实核查、叙事设计、视觉指导与全流程迭代把控。---## 我的工作### 一、选题策划与叙事设计**发现选题**该事件最初在短视频平台传播，我判断其具备互动叙事潜力——有明确的时间线（卫星发现→长途奔赴→寻找石料→修复→官方回应）、有视觉冲击力（戈壁地貌、巨幅标语、卫星影像对比）、有情感张力（一个人的自发守护）。适合用 H5 形式将观众拉入故事，而非被动阅读。**设计 8 幕叙事弧线**| 幕 | 场景 | 情绪功能 ||----|------|----------|| P0 启动 | GPS 信号 → 标题浮现 | 奠定纪实基调，制造仪式感 || P1 卫星发现 | 3D 地球缩放 → 异常检测 | 好奇：发生了什么？ || P2 奔赴戈壁 | 地图路径穿越 2168 公里 | 共情：一路向西的艰辛 || P3 寻石记 | 戈壁地表寻找匹配石料 | 专注：必须找到对的那块 || P4 大地修复师 | 24 趟运输逐像素填充伤痕 | 满足：一点一点修回来 || P5 卫星确认 | 23 天后再过顶扫描 | 闭环：修复被太空看见 || P6 回响 | 评论区气球 + 时间线叙事 | 感动：事件的社会反响 || 终章 | 修复后航拍海报 | 升华：守护不止一人 |每幕的交互手势也遵循递进逻辑：观看 → 缩放 → 滑动 → 拖拽 → 滑动 → 等待 → 翻阅，让操作难度随故事深入逐步增加。### 二、事实核查与内容修正项目自带的 `Constants.js` 中预设了 5 条时间线事件，我在开发前对其逐一核实：| 预设事件 | 核实结果 | 处理方式 ||----------|----------|----------|| 哈密市相关部门立案调查 | ✅ 搜狗搜索确认：官方已介入，肇事者被找到 | 保留 || 标语纳入不可移动文物普查 | ✅ 确认：「哈密市第四次全国文物普查新发现不可移动文物名录」 | 保留 || 多家媒体跟进报道 | ✅ B站、微博、搜狐等均有报道 | 保留 || 越野爱好者自发组织巡逻队 | ❌ 未搜到任何相关报道 | 替换为「哈密市推动戈壁生态保护立法」（有搜狗搜索结果支撑） || 「每一个你」号召性结尾 | — 设计性内容 | 保留 |同时确认了核心事实：修复者真实身份为视频博主（项目中的「路遥他们」为艺术化处理）、破坏原因为开车漂移碾压、确实跨越 2000+ 公里、确实使用了卫星地图对比。**尝试采集真实网友评论**为 P6「回响」场景的评论区气球效果，我尝试通过 B站 API、搜狗、微博、抖音、知乎、网易、QQ 新闻七个渠道抓取真实网友评论。由于各平台反爬机制，仅从 B站 API 获取到 2 条有效评论（「那个排字的提手旁是不是被破坏了两个地方」「[星星眼]」），量级不足以支撑场景。最终确认该环节需人工从平台评论区手动采集。### 三、视觉与交互指导我对 AI 生成代码的迭代反馈记录（代表性实例）：**P3「寻石记」三次重做**- 第一版：6 个 CSS 圆角 div 堆砌的石头，像塑料块。反馈：「石头建模太假了，做真实漫画风格」→ AI 重写为 Canvas 2D 手绘，含不规则多边形、交叉排线阴影、矿物脉纹、质感点- 第二版：拖拽失灵。反馈：「石头无法拖动对比」→ 定位到 canvas 子元素拦截了事件，修复为 `pointer-events:none` + `e.currentTarget`- 第三版：背景死黑。反馈后定位到正则删除 updateDots 函数时留下残留代码 `);}` 打断 JS 执行链**P4「大地修复师」文案核实**- AI 生成了 6 段里程碑叙事，我带质疑追问「文字来源是哪里？」→ AI 承认全系虚构 → 我要求搜索真实报道 → 搜狗搜索确认了 4/5 的关键事实，1 条无出处被替换**跨场景一致性把控**- 删除 P3 中金色进度圆点（喧宾夺主）- 统一 P3 选择前告知规则（为什么必须选黑褐色），而非选错后才说教- P4 中缀叙事文字添加「路遥他们」四字使人物线连贯- 所有场景使用统一的 gravelBlack / gobiSand / signalGreen 色板### 四、工具协作模式| 环节 | 工具 | 我的角色 ||------|------|----------|| 代码生成 | Codex CLI（GPT 模型） | 提出结构化需求，控制技术方向 || 视觉验证 | Chrome 浏览器 | 逐场景测试，发现问题后描述给 AI || 事实核查 | 搜狗搜索、B站 API | 验证 AI 预设内容的真实性 || 文件管理 | PowerShell 命令行 | 备份、打包、编码修复等辅助操作 |典型工作循环：提出设计需求 → AI 生成单文件 HTML → 浏览器打开测试 → 发现问题后用自然语言描述反馈 → AI 定位并修复 → 再次测试 → 迭代至满意。最多一个场景（P3）经历了 4 轮迭代。### 五、项目产出- 5 个完整交互场景（P0-P4），2 个框架就绪场景（P5-P6）- 累计约 4000 行代码（含 HTML/CSS/JS）- 12 张素材图片的视觉资产管理- 可运行的单文件原型，无需构建工具---## 技能收获通过这个项目，我实践了：-
